# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

We take the security of The Coding Ledger seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via one of these methods:

- Email: solomon.mboni@example.com
- GitHub Security Advisory: Use the "Report a Vulnerability" button in the Security tab

### What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- Acknowledge receipt within 48 hours
- Provide initial assessment within 72 hours
- Keep you informed of progress

### Disclosure Policy

- We will investigate all reported vulnerabilities
- We will work with you on a fix before public disclosure
- We will credit you in the security advisory (unless you prefer to remain anonymous)
- We ask that you give us reasonable time to address the issue before public disclosure

Thank you for helping keep The Coding Ledger secure!
